Lesson 02 · Carve the building into rooms

Subnets &
Availability Zones

Now you have the building (the VPC) and its floor space (the CIDR) 10.0.0.0/16. Time to carve it into rooms — subnets — and decide which rooms have a door to the street (public) and which are locked interior rooms (private).

Which tier goes where·High availability across AZs·Public vs private is routing
Mental model

One room in one building.
Public door or not.

A subnet is a room in one specific building (AZ). "Public vs private" is just whether that room has a door to the street. You put each tier of your app in the kind of room it deserves, and you duplicate every tier across two or three buildings so losing one building doesn't lose the app.

03
What a subnet actually is

A CIDR slice pinned to ONE AZ

A subnet takes a chunk of the VPC's address range and binds it to a single AZ. A subnet cannot span AZs. If you want your app in three AZs, you need three subnets.

VPC 10.0.0.0/16
├─ subnet 10.0.0.0/20 → us-east-2a (4,096 addresses)
├─ subnet 10.0.16.0/20 → us-east-2b
└─ subnet 10.0.32.0/20 → us-east-2c
The routing fact, not a flag

Public vs private is a
routing decision

Public subnet

Route table sends 0.0.0.0/0 → IGW

↔ Internet

Private subnet

Route table has no 0.0.0.0/0 → IGW

No direct internet

There is no checkbox that says "public = true." Privacy is emergent from the route table. Remove the IGW route and that exact same subnet becomes private.

05
Map subnets to your app tiers

Only the load balancer is public

public subnets

Internet-facing only

ALB (load balancer) · NAT Gateway (for private egress)

private subnets · app tier

Your containers

ECS Fargate — Rails web tasks, Sidekiq workers. No public IP. Reachable only via ALB.

private subnets · data tier

Your data

RDS Postgres, ElastiCache Redis. Reachable only from app tier. Never public.

Rails analogy: ALB is your public controller endpoint. Sidekiq and Postgres never accept a request straight from the internet — only from inside the trusted boundary.
Availability Zones are failure domains

Spread across AZs to survive

An AZ is a separate datacenter. AWS designs them so a power/cooling/network failure in one AZ doesn't affect another. Run each tier in at least two AZs.

AZ-a (alive)

ALB node
web task ×1
RDS primary
Redis node
ONE AZ
FAILS

AZ-b (dead)

ALB node ✗
web task ×1 ✗
RDS standby ✗
Redis node ✗

If AZ-a dies: traffic + database fail over to AZ-b, app stays up. Single-AZ is the classic rookie outage — everything in one building, that building fails, you're fully down.

Carving the /16 sensibly

Use /20 subnets — 4,096 addresses each

A 10.0.0.0/16 VPC can hold 256 non-overlapping /20 subnets. Each /20 has 4,096 addresses — plenty for Fargate tasks (one IP each), with room to scale. Don't over-optimize; address space is free. Favor generous, uniform slices over clever tight ones.

The failure you're avoiding: a subnet so small that scaling out fails with "insufficient free IP addresses." Use /20.

AWS reserves 5 addresses per subnet

Network address, VPC router, DNS server, future use, broadcast. At /20 this is negligible. At /28 (16 addresses), you lose 31% to reserves = only 11 usable.

  • Network address — first IP (e.g. 10.0.1.0 in a /24)
  • VPC router — second IP (e.g. 10.0.1.1), gateway to other subnets
  • DNS server — third IP (e.g. 10.0.1.2), AWS internal DNS
  • Future use — fourth IP reserved by AWS for future features
  • Broadcast address — last IP (e.g. 10.0.1.255 in a /24)
The subtle exposure knob

map_public_ip_on_launch

For public subnets

Turn ON — so ALB and NAT get public addresses.

For private subnets

Keep OFF (default) — no unintended public IPs.

Getting this backwards is a quiet way to hand a public IP to something you meant to keep private. A task with an unexpected public IP, plus a routing mistake, = exposed. Treat "on" as a deliberate choice for genuinely public subnets only.

The Terraform

Public + private subnets, two AZs

resource "aws_subnet" "public_a" {
  vpc_id            = aws_vpc.main.id
  cidr_block        = "10.0.0.0/20"       # /20 slice of the /16
  availability_zone = "us-east-2a"        # pinned to ONE AZ
  map_public_ip_on_launch = true  # ON for public
  tags = { Name = "public-2a" }
}

resource "aws_subnet" "private_a" {
  vpc_id            = aws_vpc.main.id
  cidr_block        = "10.0.128.0/20"
  availability_zone = "us-east-2a"
  map_public_ip_on_launch = false # OFF for private (default)
  tags = { Name = "private-2a" }
}

Notice: nothing says "public" or "private" as a property — the only real difference is the routing. A subnet becomes truly public when lesson 03 attaches a route to an Internet Gateway.
